**核心要点：**

- OpenAI 正式发布 **GPT-5.4**，这是自 GPT-5 以来最重大的单次模型升级，将通用推理、代码生成和计算机操控（Computer Use）三大能力整合至同一模型中

GPT-5.4 最重要的变化不是某项单一能力的突破，而是**能力融合架构**的成熟。在此之前，使用 OpenAI API 时往往需要为不同任务选择不同模型（推理用 o3、编码用 Codex、计算机使用用独立工具）；GPT-5.4 将这些整合为统一调用入口，大幅简化了多步骤 AI 代理的构建复杂度。"Skills" 功能本质上是将可复用提示/工具封装成可共享的原子单元，与 Claude Code 的 `/skill` 机制高度相似——这意味着 **AI 工作流的模块化** 正成为行业标配。

**背景上下文：**

### OpenAI 计划推出桌面超级应用：整合 ChatGPT、Codex 与 AI 浏览器 Atlas ⭐⭐⭐⭐⭐

**核心要点：**

- **Bloomberg** 3 月 20 日独家报道：OpenAI 正在开发桌面"超级应用（Superapp）"，将 **ChatGPT**（对话）、**Codex**（编程）、**ChatGPT Atlas**（AI 增强浏览器）三大产品整合进同一桌面客户端
- 此举被视为对 Google 将 Gemini 深度嵌入 Chrome 和 Android 生态系统的直接反制，双方的"超级应用"战略正面交锋
- Hacker News 同日头条：**OpenCode**（开源 AI 编程代理，opencode.ai）获 240 分讨论热度，被视为社区对 Codex 闭源方向的开源替代回应

**技术解读：**

桌面超级应用的战略意义在于**会话持久性与跨任务上下文共享**。当用户在浏览器中研究一个技术方案时，AI 可以直接将上下文传递给 Codex 编写代码，再由 ChatGPT 生成文档——无需反复复制粘贴。这种深度整合是纯 Web 应用难以实现的原生桌面优势。Atlas 浏览器的 AI 增强能力（类似 Arc + Perplexity 的结合）使 OpenAI 得以掌控用户浏览行为数据，进一步优化模型训练。

**背景上下文：**

此前，OpenAI 与 Pentagon 签署军事合同引发 **#QuitGPT 运动**（250 万用户取消订阅、App 卸载量单日激增 295%），桌面超级应用的发布计划或也有稳定用户留存的考量。

**开发者行动建议：**

- 关注 OpenCode（[opencode.ai](https://opencode.ai)）作为开源替代方案，特别是对隐私敏感的项目
