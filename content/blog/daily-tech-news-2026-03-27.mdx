### TeamPCP 供应链攻击第三波：Telnyx PyPI 包被 WAV 隐写木马感染，月下载量逾百万 ⭐⭐⭐⭐⭐

**核心要点：**

- **2026 年 3 月 27 日 03:51 UTC**，两个恶意版本（`telnyx` v4.87.1 和 v4.87.2）被推送至 PyPI。两个版本均已于 10:13 UTC 被隔离下架。Telnyx 官方平台及 API 基础设施未受影响，仅 PyPI 分发渠道受损
- 攻击者是 **TeamPCP** 组织——与 3 月 19 日 Trivy 漏洞（CVE-2026-33634，CVSS 9.4）、3 月 23 日 Checkmarx KICS GitHub Action 中毒、3 月 24 日 LiteLLM PyPI 投毒为同一幕后黑手。TeamPCP 通过链式凭证窃取推进攻击：上一个目标的 CI/CD 凭证成为下一个目标的入侵跳板
- 技术手法独特：恶意代码隐藏于 **WAV 音频文件中**（`hangup.wav` / `ringtone.wav`），通过 **XOR 异或混淆**将可执行 payload 编码在音频帧内，Windows 释放为自启动 `msbuild.exe`，Linux/macOS 则解码第三阶段 Python 脚本；窃取数据使用 **AES-256-CBC + RSA-4096** 加密后外传至 `83[.]142[.]209[.]203:8080`，RSA 公钥与 LiteLLM 攻击完全相同

**技术解读：**

WAV 隐写（音频隐写术）将恶意 payload 编码进合法音频文件的 PCM 帧中，绕过了大多数基于内容特征的静态分析工具——这些工具通常会扫描 `.py` 脚本中的可疑模式，却不会深度检查音频文件的帧数据。Telnyx SDK 的 `_client.py` 是包入口，植入此处意味着任何 `import telnyx` 操作都会立即触发恶意代码。TeamPCP 的链式凭证窃取模型揭示了 CI/CD 安全的系统性弱点：只要任何一个环节运行了未固定版本的扫描工具，整条链路都可能成为攻击起点。

**背景上下文：**

TeamPCP 正在与 LAPSUS$ 和新型勒索软件组织 **Vect** 协作，向勒索与数据变现方向升级。3 月 19 日至 27 日短短 8 天内连续三次成功的 PyPI 供应链攻击，标志着该组织已从机会性攻击转向针对开发工具链的系统性定向作战。

**开发者行动建议：**

- **立即降级**：确保使用 `telnyx==4.87.0` 并在 `requirements.txt` 中固定版本（附 hash 校验）
- **视为完全沦陷**：若曾运行 v4.87.1 或 v4.87.2，轮换所有可访问的 API 密钥、SSH 密钥、云凭证、数据库密码
- **Windows 排查**：检查 `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\` 中是否存在 `msbuild.exe`
- **网络层面**：屏蔽并监控对 `83[.]142[.]209[.]203:8080` 的出站流量
- **CI/CD 加固**：固定所有依赖工具版本（包括 Trivy、KICS 等安全扫描器本身）

### KubeCon Europe 2026 重磅：Kubernetes 废弃 Ingress NGINX，微软开源 AI Runway 推理 API ⭐⭐⭐⭐⭐

**核心要点：**

- Kubernetes 宣布正式**废弃 Ingress NGINX**——这是约半数云原生环境的核心网络组件，强制迁移窗口已开启，目标是 **Gateway API**（现已稳定）
- 微软在 KubeCon 阿姆斯特丹现场开源 **AI Runway**，提供"面向推理工作负载的通用 Kubernetes API"，内置 HuggingFace 模型发现和 GPU 显存适配指示器，旨在让 GPU 推理工作负载成为 Kubernetes 一等公民
- **Dynamic Resource Allocation（DRA）** 正式 GA；微软同步推出 **AKS 蓝绿节点池升级**（允许回滚），以及**预加载镜像规格（Prepared Image Spec）**，可显著降低 GPU 节点冷启动时间；跨集群网络通过托管 Cilium cluster mesh 实现

**技术解读：**

Ingress NGINX 退役是 Kubernetes 网络发展的里程碑节点：Gateway API 提供了更丰富的多租户路由语义、角色分离（基础设施运营 vs 应用团队）和标准化扩展点，解决了 Ingress 长达多年的表达力局限。AI Runway 的出现则填补了"如何在 Kubernetes 原生方式下声明和调度 LLM 推理服务"的空白——此前团队通常需要自行拼接 KubeRay、KEDA、自定义 CRD 等多个工具。

**背景上下文：**

KubeCon + CloudNativeCon Europe 2026 于阿姆斯特丹举行，本届大会的核心主题是"AI 基础设施与 Kubernetes 的融合"。GPU 感知调度、推理工作负载编排和 AI 可观测性三条线并进，Kubernetes 正从容器编排平台向 AI 原生操作系统演进。

**开发者行动建议：**

- **Ingress NGINX 用户**：立即评估 Gateway API 迁移路径，优先测试 [Kubernetes Gateway API](https://gateway-api.sigs.k8s.io/) 的 HTTPRoute 资源
- **GPU 工作负载团队**：试用 AI Runway 的 HuggingFace 集成，DRA GA 意味着可以稳定依赖其 GPU 资源切片能力
