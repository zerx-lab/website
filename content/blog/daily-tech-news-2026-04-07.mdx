### Anthropic 发布 Project Glasswing + Claude Mythos Preview：AI 扫描主流系统发现数千零日漏洞 ⭐⭐⭐⭐⭐

**核心要点：**

- **2026 年 4 月 7 日**，Anthropic 正式宣布 **Project Glasswing**，这是一项将 AI 能力用于防御性网络安全的行业联盟计划。参与方包括 Amazon Web Services、Apple、Broadcom、Cisco、CrowdStrike、Google、JPMorganChase、Linux Foundation、Microsoft、NVIDIA 和 Palo Alto Networks，另有超过 40 个关键基础设施维护机构获得了访问权限。
- Anthropic 旗下**尚未公开发布的旗舰模型 Claude Mythos Preview** 已在内部用于扫描主流操作系统和浏览器，发现了**数千个高危零日漏洞**，典型案例包括：OpenBSD 中一个存在 **27 年**的稳定性缺陷、FFmpeg 中被自动化测试遗漏数百万次的一个 **16 年**历史漏洞、以及多个 Linux 内核权限提升漏洞。
- 在网络安全基准测试中，Claude Mythos Preview 在漏洞复现任务中的得分达到 **83.1%**，远超上一代 Claude Opus 4.6 的 66.6%。
- Anthropic 为此项目承诺提供 **1 亿美元**模型使用配额，并向开源安全组织直接捐款 **400 万美元**。模型对公众的访问受到严格限制，原因是 Anthropic 已私下向美国政府官员发出警告：Mythos 的能力可能在 2026 年内使大规模 AI 驱动的网络攻击更容易发生。

**技术解读：**

Claude Mythos Preview 在网络安全领域实现了一个关键跨越：从"辅助人类找漏洞"升级为"自主闭环发现、复现并构造 PoC"。83.1% 的漏洞复现基准得分意味着 Mythos 可以在几乎没有人工干预的情况下将一个漏洞描述转化为可执行的攻击代码，这是 AI 安全工具史上前所未有的自主程度。Project Glasswing 的双重逻辑值得关注：Anthropic 选择在限制公开访问的前提下先向防御方（OS 厂商、安全公司）提供工具，试图缩短补丁窗口，在攻击方利用漏洞之前完成修复。

**背景上下文：**

这一发布恰好发生在昨日（4 月 6 日）Claude Code 高危漏洞 CVE-2026-33068 曝光之后。两件事并排出现，折射出 AI 安全领域的双重现实：AI 工具本身是攻击面，同时也正在成为最有力的防御工具。Claude Mythos Preview 的系统卡片（System Card）同步发布，在 HN 获得 471 分、333 条评论。

**开发者行动建议：**

- 如果你维护开源基础设施项目，可关注 Anthropic 的 Project Glasswing 申请通道
- 将 AI 辅助代码扫描纳入 CI/CD 安全流水线的长期规划
- 关注 Linux Foundation 和各 OS 发行版即将发布的安全公告（Glasswing 漏洞批量披露已在进行）

### Z.ai 发布 GLM-5.1：SWE-bench Pro 58.4 分超越 GPT-5.4 与 Claude Opus 4.6，持续自主运行 8 小时 ⭐⭐⭐⭐⭐

**核心要点：**

- **2026 年 4 月 7 日**，北京 Z.ai（智谱 AI）发布旗舰模型 **GLM-5.1**，在 **SWE-bench Pro** 上以 **58.4 分**成为全球最高分，超越 GPT-5.4（57.7）、Claude Opus 4.6（57.3）和 Gemini 3.1 Pro（54.2）。这是中国 AI 模型首次在该基准上超越所有美国头部模型。
- GLM-5.1 的核心突破在于**长时域任务（Long-Horizon Tasks）**：模型可在单个任务上连续自主工作长达 **8 小时**，形成"实验—分析—优化"闭环，在代表性场景中完成 **655 次迭代优化**，能够从头构建完整的 Linux 桌面系统。
- 其他关键基准：**NL2Repo 42.7**（最高）、**Terminal-Bench 2.0：63.5–66.5**、**CyberGym：68.7**（最高）、**BrowseComp：68.0–79.3**、**MCP-Atlas：71.8**。
- 架构上，GLM-5.1 是 GLM-5 的后训练升级版：744B 参数 MoE（每 Token 激活 40B），200K 上下文窗口，通过重新定向的强化学习管道提升了工程执行能力，而非改变基础架构。
- 已通过 OpenRouter 上线，模型 ID 为 `z-ai/glm-5.1`；在 NVIDIA NIM 平台有对应 `glm-5` 版本。

**技术解读：**

GLM-5.1 在 SWE-bench Pro 上超过美国模型 0.7 分，看似微小，但在当前前沿模型分数高度集中的区间，这一差距具有统计显著性。更值得关注的是其在 CyberGym（网络安全 CTF 场景）和 Terminal-Bench（命令行自主执行）上的领先，这与其 8 小时自主运行能力高度一致——GLM-5.1 明显针对"深度工程执行"场景进行了专项优化，而非追求通用推理广度。这与 Claude Opus 4.6 和 GPT-5.4 在 GPQA-Diamond 和 AIME 2026 上仍保持领先形成互补。

**背景上下文：**

此次发布正值 OpenAI/Anthropic/Google 三巨头宣布联合对抗中国模型"对抗性蒸馏"行动之日，形成强烈对比——一边在技术评测上超越，一边在 IP 保护上被针对。GLM-5.1 在 HN 获得 375 分、107 条评论，是今日 HN 热度最高的技术发布。

**开发者行动建议：**

- 在复杂工程 Agent 场景（长链重构、CI/CD 优化流水线）中评测 GLM-5.1 与 Claude Opus 4.6 的实际差异
- 通过 OpenRouter 调用 `z-ai/glm-5.1` 进行快速对比测试
- 对于对 GPQA/AIME 类推理有高要求的场景，Claude Opus 4.6 仍是更优选择

### OpenAI/Anthropic/Google 三巨头联手：通过 Frontier Model Forum 对抗中国"对抗性蒸馏" ⭐⭐⭐⭐

**核心要点：**

- **2026 年 4 月 7 日**，OpenAI、Anthropic 和 Google 宣布通过 **Frontier Model Forum**（2023 年由三家公司与微软联合创立）共享情报，系统性对抗中国 AI 公司以"对抗性蒸馏"方式侵犯其知识产权的行为。
- 点名的中国公司包括 **DeepSeek、Moonshot 和 Minimax**。"对抗性蒸馏"指通过大规模调用美国 AI 模型、收集输出结果，再用于训练成本更低的本地模型，绕过正常研发周期。
- 协作机制类似网络安全领域的威胁情报共享：各公司交换检测方法、攻击模式特征，互相通报发现的恶意调用行为。美国当局估计该行为每年给美国 AI 实验室造成**数十亿美元**损失。
- OpenAI 此前已就 DeepSeek "采用越来越复杂的方法"从其模型提取数据的问题向美国国会发出警告。

**为什么重要：** AI 知识产权保护正式进入"行业联防"阶段，美国主要 AI 公司首次绕开个体商业竞争，在 IP 防护层面形成统一战线。这将影响中国 AI 公司的基础能力获取路径和产品迭代速度。

### Google Gemini Embedding 2：首个原生统一跨模态嵌入模型 ⭐⭐⭐

Google DeepMind 发布 **Gemini Embedding 2**，这是首个能将文本、图像、视频、音频和文档原生映射到**同一语义空间**的嵌入模型，支持超过 100 种语言，无需专门的单模态嵌入器。对于构建多模态 RAG 系统的开发者，这意味着跨模态检索可以共用同一索引，大幅简化系统架构。

**为什么重要：** 多模态统一嵌入是 RAG 和语义搜索从文本专用走向"任意格式"的关键基础设施组件，Gemini Embedding 2 是目前该类别中能力覆盖最广的公开模型。

### Google 开源多 Agent 编排框架 Scion ⭐⭐⭐⭐

Google 在 HN 发布并开源 **[GoogleCloudPlatform/scion](https://github.com/GoogleCloudPlatform/scion)**，这是一个面向深度 Agent 的实验性多 Agent 编排框架，被 Google 定位为 "Agent 操作系统" / "Agent Hypervisor"（HN 138 分，43 评论）。

**核心设计：** 每个 Agent（Claude Code、Gemini CLI、Codex 等）在独立容器中运行，拥有隔离的身份、凭据和独立的 git worktree，支持本地、远程 VM 和 Kubernetes 多种运行时。框架采用"少即是多"策略——Agent 通过动态学习 CLI 工具实现协调，让模型自身决定如何分工，而非硬编码编排模式。Google 随项目发布了一个用 Agent 群体协作解谜的游戏"Relics of the Athenaeum"作为参考实现。

**为什么重要：** Scion 与微软 Autogen、LangGraph、CrewAI 直接竞争，但以 Google Cloud 原生支持和 Agent 隔离作为差异化。当前状态为早期实验性项目，本地运行相对稳定，Kubernetes 支持仍有粗糙之处。

- 来源: [InfoQ 报道](https://www.infoq.com/news/2026/04/google-agent-testbed-scion/)
- 来源: [GitHub – GoogleCloudPlatform/scion](https://github.com/GoogleCloudPlatform/scion)

### Cloudflare 加速后量子安全路线图：2029 年全平台完成，量子破密门槛意外降低 ⭐⭐⭐⭐

Cloudflare 发布博客宣布将整个平台完全迁移至后量子安全的目标从 2031 年提前至 **2029 年**，并详细公布了里程碑时间表（HN 254 分，82 评论）。

**加速原因：** Oratomic 公司最新研究将破解 P-256 椭圆曲线所需的量子比特估算压缩至约 **10,000 个**，比此前预期低得多。Google 据此修订了自身的量子算法，使"Q-Day"（量子计算机能实际破解现有加密的时间节点）比原先预期更近。

**里程碑：**
- 2026 年中：支持 ML-DSA 后量子认证用于源站连接
- 2027 年中：终端用户至 Cloudflare 的连接使用 Merkle Tree Certificates
- 2028 年初：Cloudflare One SASE 套件全面后量子认证
- 2029 年：全平台完成

**成本：** 所有升级对 Cloudflare 客户**免费**，包括免费计划用户。

**为什么重要：** 后量子加密迁移不再是"未来的事情"——前沿研究表明时间窗口比多数人预期的更短。对依赖 TLS/PKI 的后端服务，现在就需要规划迁移评估。

- 来源: [Help Net Security 报道](https://www.helpnetsecurity.com/2026/04/07/cloudflare-post-quantum-authentication/)
- 来源: [SiliconANGLE 报道](https://siliconangle.com/2026/04/07/cloudflare-fast-tracks-post-quantum-rollout-new-research-puts-encryption-notice/)
