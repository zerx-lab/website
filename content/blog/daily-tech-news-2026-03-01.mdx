## 🔥 今日焦点

### Chrome 发布 WebMCP 早期预览：每个网站都将成为 AI 代理的结构化工具 ⭐⭐⭐⭐⭐

**核心要点：**

- Google Chrome 团队发布 **WebMCP 早期预览**（Early Preview Programme），这是让浏览器与 AI 代理直接通信的新 W3C 标准。目前可在 Chrome 146 Canary 中通过 Flag 启用，Chrome 146 稳定版预计 **3 月 10 日前后**正式发布
- WebMCP 提供两套 API：**声明式 API**（Declarative API）直接通过 HTML 表单定义标准操作；**命令式 API**（Imperative API）处理需要 JavaScript 的复杂动态交互。网站通过 `navigator.modelContext` 将自身功能以结构化 JSON Schema 暴露给 AI 代理
- 相比现有"截图 → 视觉模型猜测点击位置"的方式，WebMCP 带来三项关键改进：**延迟降低**（无需上传截图）、**准确率接近 100%**（结构化 JSON 代替模糊 DOM 操作）、**成本大幅降低**（文本 Schema 远比高分辨率图像便宜）
- **安全设计**：WebMCP 采用"权限优先"机制，AI 代理无法绕过浏览器直接执行操作，敏感动作（如提交订单、转账）会弹出用户确认对话框，Chrome DevTools 也将新增专属调试面板

**技术解读：**

WebMCP 代表了 AI 代理与 Web 交互范式的根本性转变。此前，所有主流 AI 浏览器（OpenAI Operator、Anthropic Computer Use、Gemini）都依赖视觉模型"猜"界面——这导致约 30% 的操作失败率和大量 GPU 浪费。WebMCP 让网站"主动告知" AI 自己能做什么，就像 REST API 定义端点一样清晰。

值得注意的是，WebMCP **并不是 Anthropic MCP 的替代品**：Anthropic MCP 是后端协议（AI 平台与服务商之间的通信标准），WebMCP 是纯前端浏览器标准，两者互补而非竞争。

**背景上下文：**

WebMCP 由 Google 和 Microsoft 工程师共同设计，在 W3C 机器学习社区小组中开发。截至 2026 年 2 月，仅 Chrome Canary 支持，Edge 支持在路上，Firefox 和 Safari 尚未表态。行业分析师预计 Google I/O 将是更广泛发布的关键节点。

**开发者行动建议：**

- 现在就可以为网站添加 WebMCP 元数据，提前让你的网站成为"AI 就绪"网站
- 在 Chrome 146 Canary 中启用 `chrome://flags/#webmcp` 测试你的实现
- 关注 Chrome Web Store 上已上架的 **Model Context Tool Inspector** 扩展，可用于调试 Schema 错误

### 字节跳动开源 DeerFlow 2.0：从零重写的 SuperAgent 跃居 GitHub 热榜第一 ⭐⭐⭐⭐⭐

**核心要点：**

- 字节跳动于 **2026 年 2 月 27 日**开源 DeerFlow，并在 **2 月 28 日**发布彻底重写的 **2.0 版本**，随即登顶 **GitHub Trending 第一名**，单日获得 692+ 星标，当前累计超过 **22,900 stars**
- DeerFlow 2.0 定位为"SuperAgent Harness"——不是一个框架，而是**一个完整的、开箱即用的 AI 代理运行环境**。基于 **LangGraph + LangChain** 构建，内置文件系统、跨会话记忆（TIAMAT Cloud Memory）、沙箱执行（Docker/Kubernetes）、技能系统和子代理编排
- 内置技能涵盖：网络研究、代码执行、报告生成、幻灯片制作、图像/视频生成；支持通过 **MCP 服务器**和自定义 Python 函数扩展新工具
- 三种部署模式：**本地开发**（直接运行）、**Docker 单服务器生产**、**Kubernetes 多节点集群**；企业级功能包括 RBAC 权限控制、审计日志、凭据管理和安全通信，MIT 许可证开源

**技术解读：**

DeerFlow 2.0 的核心设计理念是"电池已包含"（Batteries Included）——区别于 AutoGPT、CrewAI 等需要大量配置的框架，DeerFlow 提供了一个已经连线好所有组件的运行环境。用 LangGraph 的状态机管理复杂的多步骤任务，确保代理在长时间运行的任务中不会"失忆"或"跑偏"。

最值得关注的是沙箱执行设计：代码默认在 Docker/K8s 容器中运行，与主机隔离，这是企业级部署的基本要求，也是早期代理框架的普遍痛点。

**背景上下文：**

代理框架市场在 2026 年 2 月迎来"寒武纪大爆发"，单月就有五个主要框架发布或引爆。DeerFlow 1.x（原名 Deep Research Framework）已积累用户基础，2.0 从头重写说明字节跳动对代理框架有长远打算。行业分析师预测 2027 年将出现整合，最终形成 2-3 个主导框架。

### WiFi DensePose：无需摄像头，用 WiFi 信号实时追踪人体姿态 ⭐⭐⭐⭐⭐

**核心要点：**

- 开源项目 **wifi-densepose**（GitHub: ruvnet/wifi-densepose）当前拥有 **17,067 stars**，在 GitHub Trending 持续高位。项目将卡内基梅隆大学 2023 年论文《DensePose From WiFi》工程化为可直接部署的生产级实现
- 核心原理：分析 **信道状态信息（CSI）**——WiFi 信号在遇到人体时的相位/幅度变化模式。将 30 个频率 × 3 发射 × 3 接收的数据组成两个 150×3×3 张量，输入改造后的 DensePose-RCNN 模型，输出 24 个人体区域的 UV 坐标
- 关键指标：**准确率 94.2%**（与摄像头方案相当）、**延迟 &lt;50ms**、支持同时追踪 **最多 10 人**、可穿透墙壁。提供 REST API、PyPI 包（`pip install wifi-densepose`）和 Docker 镜像
- **硬件要求**：标准消费级 WiFi 路由器无法暴露 CSI 数据，需要 ESP32-S3 或专用研究网卡（如 Intel 5300 NIC）才能获得原始 CSI 数据

**技术解读：**

WiFi DensePose 的核心价值在于"无画面隐私"：系统输出的是姿态坐标（关节点位置），而非图像或人脸数据，天然符合 GDPR 等隐私法规。这使得原本需要摄像头的应用场景——跌倒检测、康复监测、智能家居、搜救——可以在隐私敏感环境中部署。

当前最大障碍是硬件门槛：ESP32-S3 约 $5-10，但研究级网卡价格较高。随着支持 CSI 的 WiFi 6/6E 设备逐渐普及，这一门槛有望在未来 1-2 年内大幅降低。

**背景上下文：**

学术研究到生产级开源工具的转化时间越来越短——CMU 2023 年的论文在约 3 年内就有了生产级实现。这反映了 AI 开发者社区将论文"工程化"的速度正在加快，也让非视觉 AI 感知技术的应用门槛大幅降低。

**开发者行动建议：**

- 如果你的项目涉及人员检测/追踪但有隐私顾虑，WiFi DensePose 是摄像头方案的可行替代
- 搜救、老年看护、智能家居场景最有商业落地潜力
- 先用 ESP32-S3 + Docker 搭建概念验证，验证场景可行性再评估研究级硬件

**相关链接：**

- 学术论文: [DensePose From WiFi (arXiv)](https://arxiv.org/abs/2301.00250)
