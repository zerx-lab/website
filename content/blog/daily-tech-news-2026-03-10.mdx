### Anthropic 起诉五角大楼：拒绝无限制军事用途，OpenAI 和 Google 员工联名声援 ⭐⭐⭐⭐⭐

**核心要点：**

- **Anthropic** 于3月9日正式对特朗普政府提起**两项联邦诉讼**，起诉地点分别为加州北区联邦法院和华盛顿特区联邦上诉法院。起因是美国国防部将其列为"供应链风险"——这一标签在历史上仅用于**外国敌对势力**，Anthropic 是**首家**被如此对待的美国本土公司
- 矛盾根源：Anthropic 要求国防部承诺，不将 Claude 用于**大规模监控美国公民**或**自主致命武器系统开发**。五角大楼则坚持"所有合法用途均可"，双方谈判破裂。2025年7月，Anthropic 与国防部签署了2亿美元合同，是首家将 AI 部署到机密网络的实验室
- 最新进展：**逾900名**科技从业者联署公开信《我们不会被分裂》，其中包括近100名 **OpenAI** 和近800名 **Google DeepMind** 员工（以个人名义参与）；Google 首席科学家 **Jeff Dean** 等多名重量级研究员提交法庭之友意见书，指出将 Anthropic 列黑名单将损害整个美国 AI 产业竞争力
- 经济风险：Anthropic 警告此举可能削减其2026年**数十亿美元**营收，目前公司估值**3800亿美元**，超过500家企业客户年付费超百万美元。GSA 已终止 Anthropic 的"OneGov"合同，联邦三权机构全面停用 Claude

**技术解读：**

这场法律战的核心不是合同纠纷，而是一个关乎整个 AI 行业的根本问题：**私人 AI 公司能否对政府客户设定伦理边界，而不被当作国家安全威胁对待？** Anthropic 的诉求并非迫使政府继续合作，而是防止联邦机构因政策分歧将企业列入黑名单，这将为所有坚持 AI 安全原则的公司树立关键先例。值得关注的是，同期 **OpenAI** 与国防部迅速达成协议（放弃类似限制条款），但随后承认其声明"显得草率和机会主义"，目前正重新谈判部分条款。

**开发者行动建议：**

- 关注此案判决走向：若 Anthropic 败诉，意味着政府合同实际上要求 AI 公司放弃所有使用限制——将深刻影响企业选择 AI 服务商的决策
- 企业 AI 合规团队应重新评估政府采购合同中的技术使用条款，特别是涉及"所有合法用途"的模糊表述

### Yann LeCun 的 AMI Labs 完成10.3亿美元种子轮：押注"世界模型"颠覆 LLM ⭐⭐⭐⭐⭐

**核心要点：**

- 图灵奖得主、AI 教父 **Yann LeCun** 离开 Meta 后创立的 **AMI Labs**（Advanced Machine Intelligence，法语"朋友"之意）于3月10日宣布完成 **10.3亿美元种子融资**，估值 **35亿美元**，创下**欧洲史上最大种子轮**纪录
- 投资方阵容豪华：联合领投方包括 Cathay Innovation、Greycroft、Hiro Capital、HV Capital 和 **Bezos Expeditions**；战略投资方涵盖 **Nvidia**、Toyota Ventures、Temasek、三星等；个人投资者包括 Tim Berners-Lee（万维网发明人）、Mark Cuban、Eric Schmidt 等
- 核心技术路线：**JEPA（联合嵌入预测架构）**，LeCun 认为 LLM 本质上是"统计模式匹配器"，无法实现真正推理和物理世界理解。JEPA 不预测像素级或词级细节，而是学习世界运作规律的**抽象表示**，目标是构建具备持久记忆、规划能力和可控决策的 AI 系统
- 首个合作伙伴：医疗 AI 公司 **Nabla**，将优先获取技术以开发下一代医疗 AI Agent
- 团队构成：LeCun 任董事长，CEO 为 Alexandre LeBrun（Nabla 创始人），CSO 为 Meta 前研究员 Saining Xie，公司总部设于**巴黎**，并在纽约、蒙特利尔、新加坡设立分支

**技术解读：**

AMI Labs 目前**无产品、无营收**，LeCun 本人承认第一年将专注于纯研究。这是一个极端的长线押注：以10亿美元种子资金，挑战整个 LLM 范式。支持者认为，LLM 在"世界模型"（理解物理因果关系、具备空间感知）层面的局限是真实存在的；反对者则指出，Transformer 架构的涌现能力远超最初预期，世界模型能否超越 LLM 仍未可知。无论结果如何，这一赌注已成为 2026 年 AI 领域最具争议性的话题之一。

### 亚马逊强制要求高级工程师审核 AI 辅助代码：一周四次严重故障触发政策转变 ⭐⭐⭐⭐⭐

**核心要点：**

- 亚马逊本周召开全公司工程师大会，检讨近期 AI 辅助代码变更引发的多起严重事故。3月5日，一次长达**6小时的购物网站故障**导致结账、登录和商品定价全线瘫痪，一周内共发生**4次 Sev-1（最高严重级别）**事故
- 新政策：初级和中级工程师在将 AI 辅助代码变更推送到生产环境前，**必须获得高级工程师的书面批准**。亚马逊电商技术负责人 Dave Treadwell 表示，公司将"引入受控摩擦"，实施临时安全措施并投资更持久的确定性和 Agent 式保障方案
- 背景：亚马逊已在零售部门部署 **21,000 个 AI Agent**，宣称节省20亿美元成本、开发效率提升4.5倍；同时强制员工使用内部 AI 编码工具 **Kiro**，但约1,500名工程师通过内部论坛抗议，认为 Claude Code 等外部工具在多语言重构上更出色
- 内部文件显示：部分 AI 辅助变更导致的事故可追溯至2025年第三季度，其中一次是 Kiro 在 AWS 成本计算工具上**删除并重建整个环境**，导致13小时中断

**技术解读：**

这是企业大规模 AI 编码工具部署后，第一批真实生产事故和政策响应的典型案例。核心问题不在于"AI 是否会犯错"（会），而在于**现有部署流程是否足以在 AI 速度与生产安全之间建立有效缓冲**。亚马逊无法回退 AI 采用（21,000 个 Agent 已深度嵌入流程），只能在已运行的系统上增加人工审核节点——用"驾驶中修飞机"来形容毫不夸张。对所有正在或计划大规模推广 AI 编码工具的工程团队，此案例具有极高的参考价值。

**开发者行动建议：**

- 建立 AI 辅助代码的**分级审核机制**：高影响变更（核心交易流程、数据库迁移）必须经人工复核，不能完全依赖 AI 的自我评估
- 在 CI/CD 流水线中为 AI 生成的变更添加**额外的回滚保障**和监控告警
- 重新审视"两人批准"等现有安全控制是否在 AI 工具使用场景下仍然有效执行

### Cloudflare MCP Server "Code Mode"：1000 个 Token 覆盖全部 API ⭐⭐⭐⭐

**Cloudflare** 推出 MCP Server 的 **Code Mode**，重新定义开发者与 API 的交互方式：

- 仅需约 **1,000 个 token**，即可覆盖 Cloudflare 全部 API 端点。MCP Server 通过 `search()` 和 `execute()` 两个核心工具，让 AI Agent 自动发现并调用所有 API，新产品上线后无需更新工具定义
- 符合最新 MCP 规范，基于 **OAuth 2.1** 权限管控，Agent 只能访问用户明确授权的资源

**为什么重要：** Token 效率和权限细粒度是 MCP 生态的两大痛点，Cloudflare 的实现方案为其他 SaaS 平台提供了可参考的范本。

- 来源: [Cloudflare Blog - Code Mode](https://blog.cloudflare.com/code-mode-mcp/), [Cloudflare Changelog](https://developers.cloudflare.com/changelog/)

### RISC-V 性能深度分析：在现代工作负载下显著落后 x86 和 ARM ⭐⭐⭐

**Marcin Juszkiewicz** 发布深度性能测试报告 "RISC-V Is Sloooow"，在 Hacker News 引发热烈讨论：

- 测试结论：在编译、数学运算、数据库等典型工作负载下，RISC-V 硬件的性能落后于同代 x86 和 ARM 处理器**数倍乃至数十倍**
- 根本原因：RISC-V 软件生态（编译器优化、库针对性调优）仍不成熟，硬件实现多处于早期阶段，并非架构本身存在致命缺陷
- 社区讨论焦点：RISC-V 在嵌入式和定制化场景的价值不受影响；但对于通用计算，在生态成熟前仍需谨慎

- 来源: [marcin.juszkiewicz.com.pl](https://marcin.juszkiewicz.com.pl/2026/03/10/risc-v-is-sloooow/)

### 计算机科学巨匠 Tony Hoare 辞世，享年91岁 ⭐⭐⭐⭐

**Tony Hoare**（1934-2026）于近日辞世，享年91岁。Hacker News 社区今日置顶悼念。

- Hoare 是图灵奖得主（1980年），以发明 **QuickSort 算法**（1959年）和提出 **Hoare 逻辑**（程序正确性形式化验证基础）而闻名
- 他同时提出了 **CSP（通信顺序进程）**理论，这是 Go 语言并发模型的直接理论来源
- Hoare 也是"空指针引用"的发明者，他本人将其称为"我的十亿美元错误"——这一设计缺陷在数十年间造成了无数软件崩溃
- 他晚年专注于软件可靠性和形式化验证研究，对现代编程语言设计影响深远
