### PromptPwnd：AI 供应链攻击袭击 GitHub Actions，5 家 Fortune 500 公司受影响 ⭐⭐⭐⭐⭐

**核心要点：**

- 安全公司 Aikido Security 披露名为 **PromptPwnd** 的新型漏洞类别：攻击者通过在 GitHub Issue 或 PR 中嵌入恶意提示词，操控 CI/CD 流水线中的 AI 代理执行特权操作——窃取 Secrets、修改仓库数据、破坏供应链完整性
- 受影响的 AI 工具涵盖 **Google Gemini CLI、Claude Code、OpenAI Codex 和 GitHub AI Inference**。Google 自己的 Gemini CLI 仓库也受影响，在 Aikido 负责任披露后 **4 天内完成修复**
- 至少 **5 家 Fortune 500 公司**的 GitHub Actions 工作流受到影响，攻击面可能远不止于此——任何将 AI 代理集成进 PR Review 或依赖更新流水线的项目均在风险范围内
- 攻击模式极为简单：**任何能提交 Issue 或 PR 的用户**（包括外部攻击者）都可触发漏洞，无需任何特权

**技术解读：**

PromptPwnd 是 AI 与 DevOps 深度融合后的必然安全裂缝。当 AI 代理被赋予"读 Issue + 执行操作"的能力时，任何写入 Issue 的人都获得了间接控制 AI 代理行为的能力。本质上，这是**授权边界的错位**：AI 代理以高权限运行，但它的"指令来源"却未经安全验证。与传统命令注入不同，提示注入无明显语法特征，难以用正则过滤。

**开发者行动建议：**

- 立即审查 GitHub Actions 工作流中是否有 AI 代理读取 Issue/PR 内容后执行操作的步骤
- Aikido 提供**免费扫描工具**检测 GitHub/GitLab 仓库中的 PromptPwnd 漏洞
- 原则：限制 AI 代理的工具集（禁止写入 Issue/PR），对所有 AI 输出执行人工审查后再运行，将 GitHub Token 访问限制在特定 IP 范围
