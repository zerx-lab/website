### Ladybird 浏览器正式采用 Rust，AI 辅助完成 25,000 行代码移植 ⭐⭐⭐⭐⭐

**核心要点：**
- 2026 年 2 月 23 日，Ladybird 浏览器创始人 Andreas Kling 宣布：项目正式将 **Rust 作为 C++ 之外的第二实现语言**，首个目标是 JavaScript 引擎 **LibJS** 的词法分析器、解析器、AST 和字节码生成器
- 移植结果令人印象深刻：约 **25,000 行 Rust 代码**，耗时仅 **约两周**——同等工作量手工完成至少需要数月
- 验证结果：test262 测试套件 52,898 条、Ladybird 回归测试 12,461 条，**零回归**；每个 Rust 解析器生成的 AST 与 C++ 版完全相同，所有字节码输出**逐字节一致**
- 此前探索过 **Swift**，但最终因 C++ 互操作性不成熟、平台支持局限于 Apple 生态而放弃；Rust 的成熟生态和内存安全保障最终胜出
- 移植工具：使用 **Claude Code 和 OpenAI Codex** 进行 AI 辅助翻译——由人类主导决策，AI 辅助实现，并经过多轮对抗性代码审查

**技术解读：**

Ladybird 的 Rust 迁移路径非常务实：LibJS 被选为首个目标，正是因为它相对自成体系，测试覆盖率高（test262 是 JavaScript 合规性测试套件的黄金标准）。

移植并非"用 Rust 重写一切"的冒进策略，而是**双轨并行**：C++ 开发继续推进，Rust 侧移通过明确定义的互操作边界逐步推进。这种策略避免了 Firefox 量子计划式的"全局暂停"风险。

AI 辅助的使用方式同样值得关注：Kling 明确表示这是"数百个小 prompt 引导 Agent"的精细化协作，而非"让 AI 自主完成所有工作"——这也印证了当前 AI Agent 的最佳实践：**人类定方向，AI 提效率**。

值得一提的背景：Linux 内核 2022 年开始接受 Rust 代码，Microsoft 持续增加 Windows 组件中的 Rust 比例，Android 安全团队将新代码中内存安全漏洞减少归功于 Rust 的引入——浏览器引擎转向 Rust，是大势所趋。

**背景上下文：**

Ladybird 是目前最受期待的独立浏览器引擎项目，目标是建立一个完全独立于 Chromium（Google）和 WebKit（Apple）的浏览器引擎。2026 年 Linux 和 macOS Alpha 版仍在计划中。Kling 称"我知道这会是个有争议的举动，但我相信这是 Ladybird 未来的正确决策"。

**开发者行动建议：**
- 关注 Ladybird 项目的 Rust 移植进展，其 Rust + C++ 互操作的工程实践值得借鉴
- 对于正在评估"C++ 项目引入 Rust"的团队，Ladybird 的双轨策略提供了可参考的路径
- 从这次 AI 辅助移植中学习：在明确测试基准（字节码输出完全一致）的前提下使用 AI 辅助翻译，是减少回归风险的关键

### AI 辅助编写 FreeBSD Wi-Fi 内核驱动：一次真实的 Agent 工程实验 ⭐⭐⭐⭐

**核心要点：**
- 开发者 Vladimir Varankin 发布博文，记录了使用 AI Agent 为 MacBook 2016（Broadcom BCM4350 芯片）编写 FreeBSD Wi-Fi 驱动的完整实验历程
- 在 Hacker News 获得 **127 分、高质量技术讨论**
- 初次尝试：让 **Claude Code** 将 Linux 的 brcmfmac 驱动移植到 FreeBSD，参照 iwlwifi 驱动和 LinuxKPI 兼容层——结果是 Agent 持续产生 panic，代码量越来越大，驱动始终无法工作
- 转变策略：改用 **Pi Agent**，从头开始，只给一个规格文档——要求 Agent 在编码前先提出所有重要设计决策，结果显著改善
- 关键洞察：在 `AGENTS.md` 中明确要求"记录所有决策点"，使 Agent 的工作更可追溯和可控

**技术解读：**

这个实验揭示了当前 AI Agent 编写系统级代码（如内核驱动）的真实局限：当任务足够复杂、调试信号不明确（内核 panic 而非友好错误信息）时，Agent 容易陷入"越改越乱"的循环。作者的解决方案是两步：首先**让 Agent 规划和提问**（而非直接开始写代码），其次**通过文档锚定决策**（而非让 Agent 自由发挥）。

对于使用 AI Agent 处理复杂工程任务的开发者，这是一份极有价值的一线报告：**明确的规格 + 提前暴露设计决策 + 文档化约束**，是让 Agent 在复杂任务中保持正轨的关键。

**开发者行动建议：**
- 在复杂 Agent 任务开始时，先让 Agent 列出需要做的所有决策，再开始写代码
- 使用 `AGENTS.md` / `CLAUDE.md` 文档化决策约束，而非依赖 prompt 传递上下文
- 阅读原文：即使对 FreeBSD 不感兴趣，这也是一篇关于 AI Agent 工程实践的优质案例研究
