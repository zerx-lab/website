## 🔥 今日焦点

### Anthropic 被五角大楼列为"供应链风险"，科技行业集体发声 ⭐⭐⭐⭐⭐

**核心要点：**

- **3 月 6 日**，美国国防部（DoD）正式将 Anthropic 认定为**"供应链风险"**，禁止其参与联邦国防合同。Anthropic CEO Dario Amodei 确认已收到该认定，并表示公司"别无选择"，将通过法律途径提出质疑
- Google 在国防部认定发布次日宣布：**该认定不影响 Anthropic 在非国防相关项目上的合作**，Claude 模型将继续在 Google Cloud Vertex AI 平台提供服务。微软和亚马逊随后发表类似声明，均表示继续向云客户提供 Anthropic 技术（国防业务除外）
- 跨公司联署信迅速扩散：一封题为"我们不会被分裂"的公开信在数日内从数百人增至近 **900 名签署者**，其中约 100 名来自 OpenAI，近 800 名来自 Google。此外另有数百名科技工作者联署呼吁国防部撤销对 Anthropic 的认定，签署者涵盖 OpenAI、Salesforce、Databricks、IBM 和 Cursor 等公司员工
- OpenAI CEO Sam Altman 公开表态支持 Anthropic，称"我个人认为五角大楼不应该用《国防生产法》威胁这些公司"

**技术解读：**

此次事件的核心矛盾在于：AI 公司为商业竞争和安全研究设定的"红线"，与政府机构对 AI 能力的无限制需求之间的冲突。Anthropic 的 Claude 模型以安全性为核心设计理念，公司长期维持不直接参与致命武器自主决策的红线。DoD 的"供应链风险"认定意味着任何将 Anthropic 纳入供应链的联邦承包商可能面临合规风险——这对依赖 Claude API 的企业客户产生了连锁影响。Google、微软、亚马逊的声明本质上是在为客户提供**合规路径说明**，而非直接对抗国防部。

**背景上下文：**

Anthropic 近期融资超 100 亿美元，估值达 3500 亿美元。随着其编程模型（Claude Code 等）帮助公司快速缩小与 OpenAI 的营收差距，其商业价值已足够庞大，使得"供应链风险"认定对整个科技生态产生实质性震动。MCP（模型上下文协议）已被 OpenAI、微软、Google 广泛采纳并由 Linux 基金会管理，这进一步凸显了 Anthropic 在 AI 基础设施层的战略地位。

**开发者行动建议：**

- 使用 Claude API 的企业开发者应关注后续法律进展，目前 Google Cloud 和 AWS 均表示服务不受影响
- 关注 Anthropic 的 MCP 标准演进——即便商业合规环境变化，MCP 作为开放标准的生命力不受单一公司状态影响
- 此事件提示：AI 供应链风险不仅来自技术层（如昨日报道的 PromptPwnd），也来自地缘政治和监管层

### Apple MacBook Pro M5 Pro/Max 正式发布：Fusion 架构 + AI 性能提升 4 倍 ⭐⭐⭐⭐

**核心要点：**

- **3 月 3-4 日**，Apple 正式发布搭载 M5 Pro 和 M5 Max 芯片的新款 MacBook Pro（14 英寸和 16 英寸），预购于 **3 月 4 日**开启，正式发售日期为 **3 月 11 日**
- 两款芯片均采用全新 **Fusion Architecture**（融合架构）——首次将两枚第三代 3nm 芯片通过先进封装技术键合成单一 SoC，这是 Apple Silicon 单芯片设计以来的重大架构变革。18 核 CPU（6 个超级核心 + 12 个性能核心）提供高达 **30% 的多线程性能提升**
- AI 性能大幅跃升：GPU 每个核心内置 Neural Accelerator，M5 Max 支持高达 **128GB 统一内存**（带宽 614GB/s），AI 性能较 M4 上代提升 **4 倍**，较 M1 提升 **8 倍**，可本地运行大型语言模型
- 新增 **Wi-Fi 7**（苹果自研 N1 无线芯片）、**蓝牙 6**、三个 **Thunderbolt 5 端口**（120Gbps），续航最长 **24 小时**

**技术解读：**

Fusion Architecture 将两枚硅片键合的设计让 Apple 突破了单芯片面积的物理极限，在不牺牲良品率的前提下大幅扩展了核心数量和内存带宽。对于本地 AI 推理场景，128GB 统一内存配合 614GB/s 带宽意味着可以流畅运行 700 亿参数量级（70B）的开源模型，这对注重数据隐私的开发者和研究者极具吸引力。Neural Accelerator 进入 GPU 每个核心的设计，使得 AI 推理可以与图形渲染并行，而非竞争同一资源。

### Meta Llama 4：首个原生多模态开源 MoE 模型，10M Token 上下文窗口 ⭐⭐⭐⭐

Meta 发布 **Llama 4** 系列，包含 Scout 和 Maverick 两款开放权重模型，是 Meta 首批采用**混合专家（MoE）架构**和**原生多模态**设计的模型。

- **Llama 4 Scout**：17B 激活参数 / 16 个专家，拥有业界领先的 **1000 万 Token 上下文窗口**，可在单张 NVIDIA H100 GPU 上运行，综合性能超越 Gemma 3 和 Gemini 2.0 Flash-Lite
- **Llama 4 Maverick**：17B 激活参数 / 128 个专家，在多项基准测试上超越 GPT-4o 和 Gemini 2.0 Flash，推理和编码能力可与 DeepSeek v3 媲美，但激活参数量不到其一半
- 所有模型均通过**早期融合**原生支持文本、图像和视频输入，在 200 种语言上预训练，多语言 token 数量是 Llama 3 的 10 倍
- 可通过 Hugging Face 下载，也集成在 WhatsApp、Messenger、Instagram Direct 和 Meta.AI 网站中

**为什么重要：** MoE 架构 + 原生多模态 + 10M 上下文窗口的组合，让开源模型在能力维度上首次全面追平商业闭源模型，为追求数据私有化部署的企业提供了真正可用的选项。

## 前端开发

### Angular 21 将无 Zone 变更检测设为默认，信号化时代正式开启 ⭐⭐⭐⭐

Angular 21 正式将**无 Zone 变更检测（Zoneless Change Detection）**设为默认模式。Angular 现在完全依赖 Signals（`$signal`、`$computed`）实现细粒度响应式，彻底摆脱对 `zone.js` 的依赖，运行时性能提升约 **20-30%**。

这一变化标志着 Angular 完成了从脏检查到响应式信号的完整迁移，与 Vue 的 Vapor Mode、Svelte 5 的 Runes 系统共同推动前端框架进入**信号化时代**。对于正在维护 Angular 项目的团队，需要评估 `zone.js` 的移除对现有异步代码（setTimeout、HTTP 请求、第三方库）的兼容性影响。

**为什么重要：** Angular 在三大主流框架中率先完成信号化迁移，预示着 React Compiler 主导的方向已成前端行业共识，无论使用哪个框架，开发者都应理解信号响应式的核心概念。

- 来源: [JavaScript Framework Trends 2026 - nucamp.co](https://www.nucamp.co/blog/javascript-framework-trends-in-2026-what-s-new-in-react-next.js-vue-angular-and-svelte)
