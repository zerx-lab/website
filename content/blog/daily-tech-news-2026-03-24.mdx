### LiteLLM 1.82.7 / 1.82.8 遭供应链攻击，所有凭证面临泄漏风险 ⭐⭐⭐⭐⭐

**核心要点：**

- **PyPI 上的 `litellm==1.82.7` 和 `litellm==1.82.8` 已被确认包含恶意代码**，攻击者在包内植入了 `.pth` 文件（`litellm_init.pth`，34,628 字节），该文件会在 **Python 解释器每次启动时自动执行**，无需任何 `import` 语句
- 恶意载荷收集范围极广：**SSH 密钥、Git 凭证、AWS/GCP/Azure 云密钥、Kubernetes 配置、Docker 认证文件、环境变量中的 API Key、Shell 历史记录、加密货币钱包数据、数据库密码**，全部使用 AES-256 + RSA 双重加密后上传至 `https://models.litellm.cloud/`
- 攻击向量利用了 Python `site-packages` 机制的特性——`.pth` 文件的自动执行在 Python 社区中属于鲜为人知的攻击面，CI/CD 流水线、云容器、开发机均可能受影响

**技术解读：**

这是一次精心设计的供应链攻击。传统的恶意包攻击通常需要代码显式 `import` 才能触发，而 `.pth` 文件机制绕过了这一防线：只要 Python 启动，恶意代码就会运行——哪怕你的代码一行都没有执行。攻击面之所以如此宽广，是因为 LiteLLM 常被用作 AI 代理和 LLM 调用的核心网关，使用该库的环境往往同时存储了多种 API Key 和云凭证。

受攻击最严重的场景：CI/CD 流水线（GitHub Actions、GitLab CI）中运行了受感染版本，可能已将所有 secret 泄漏给攻击者。

**背景上下文：**

LiteLLM 是目前最流行的多模型 LLM 调用统一接口库之一，在 GitHub 上拥有数万 star。此次攻击发生在 PyPI 账号安全机制完善后依然成功，表明攻击者可能获得了发布账号的控制权而非劫持构建流程。

**开发者行动建议：**

- **立即检查**：在所有环境中运行 `find $(python -c "import site; print(site.getsitepackages()[0])") -name "litellm_init.pth"` 检测是否存在恶意文件
- **立即升级**：卸载受影响版本，升级到官方已确认安全的最新版本
- **立即轮换凭证**：轮换所有在受感染环境中使用过的 SSH 密钥、API Key、云服务密钥
- **审计 CI/CD**：检查 CI/CD 日志，确认近期 build 中是否安装过受感染版本

### Wine 11 内核级 NTSYNC 重写：Dirt 3 从 110 FPS 飙升至 860 FPS ⭐⭐⭐⭐⭐

**核心要点：**

- **Wine 11 通过 NTSYNC 内核驱动彻底重写了同步处理机制**，这一驱动已合并入 Linux 6.14，直接在内核层实现 Windows NT 同步对象 API，取代了此前多年依赖的用户态 esync/fsync 补丁方案
- 实测性能提升令人震惊：**Dirt 3：110 → 860 FPS（+678%）**、《生化危机 2》：26 → 77 FPS、《无主之地：小蒂娜的奇幻之地》：130 → 360 FPS；《使命召唤：黑色行动 1》此前完全无法运行，现已可正常游戏
- Wine 11 同步完成了 **WoW64 统一层**（彻底消除 32/64 位多库依赖）、**Wayland 驱动成熟化**（支持剪贴板双向同步、拖放、正确缩放）、**Vulkan 1.4 支持**以及硬件加速 H.264 解码

**技术解读：**

NTSYNC 的核心思路是"用正确的方式实现，而非模拟"。Windows 的 NT 内核同步原语（Mutex、Event、Semaphore 等）此前只能在 Linux 用户态通过 futex 近似实现，每次上下文切换都有额外开销。NTSYNC 在内核里直接提供等价语义，消除了数层翻译开销。对于大量依赖多线程同步的游戏（恰恰是最主流的 3A 游戏），这一改变是质的飞跃。

**背景上下文：**

Wine 是 Linux 运行 Windows 程序的核心兼容层，Valve 的 Proton（Steam Deck 的技术底座）正是基于 Wine 构建。Wine 11 被评为"自 Proton 让 Linux 游戏成为可能以来最重要的 Wine 版本"。随着 Linux 6.14 集成 NTSYNC，发行版用户无需额外打补丁即可享受这一性能提升。

**开发者行动建议：**

- 升级到 Linux 6.14+ 内核以启用原生 NTSYNC 支持
- 更新至 Wine 11 并在游戏性能问题中测试效果
- 针对 Linux 的 Windows 兼容层应用开发者可关注 WoW64 统一带来的 32 位支持改善
