### Google 发布 Gemma 4 开源模型：Apache 2.0 授权，31B 全球第三 ⭐⭐⭐⭐⭐

**核心要点：**

- Google 于 2026 年 4 月 2 日发布 Gemma 4 系列，共四款：边缘端 **E2B / E4B**（支持手机、树莓派等设备，原生音频输入，128K 上下文），以及消费级 GPU **26B MoE** 和 **31B Dense**（256K 上下文）。
- 首次采用 **Apache 2.0 许可证**，可完全免费商用，权重通过 Hugging Face、Ollama、Kaggle、LM Studio、Docker 分发。
- 31B 在 Arena AI 文本排行榜位列**全球第三**（得分 1452），AIME 2026 数学 89.2%，LiveCodeBench v6 编码 80.0%，GPQA Diamond 科学知识 84.3%。

**技术解读：**

Gemma 4 基于 Gemini 3 研究成果，强调"每参数智能"——以更少参数实现更强推理。原生支持函数调用、结构化 JSON 输出与系统指令，直接面向 Agent 开发场景。E2B/E4B 的音频输入能力使端侧多模态应用门槛大幅降低；NVIDIA 同步宣布对 Gemma 4 进行 RTX 本地推理优化，与 AMD Lemonade 一起构成"本地 AI 双引擎"格局。

**背景上下文：**

Gemma 系列上一代（Gemma 3 27B）在评测中被中国竞争者超越，此次 Gemma 4 以 Apache 2.0 和更强基准正面回应。开源大模型军备竞赛从"能用"进入"生产级"阶段，开发者选择生态权重显著提升。

**开发者行动建议：**

- 通过 `ollama pull gemma4:31b` 或 Hugging Face 直接下载测试 31B 版本
- E2B/E4B 适合移动端和嵌入式 AI 应用，可替代现有云端 API 调用以降低延迟和成本
- 关注函数调用和 JSON 输出能力，可直接集成到 LangChain / LlamaIndex 等框架

### Cursor 3：从 AI 编辑器进化为多 Agent 统一工作台 ⭐⭐⭐⭐⭐

**核心要点：**

- Cursor 3 于 2026 年 4 月 2 日正式发布，核心变化是从"AI 增强 IDE"重构为**统一 Agent 工作台**，默认视图改为 Agent 编排面板。
- 支持**多 Agent 并行**：本地与云端 Agent 统一呈现在侧边栏，包括从 Slack、GitHub、Linear 等平台发起的任务；可无缝将 Agent 会话从本地迁移到云端以保持离线执行。
- 内置自研模型 **Composer 2**，集成 LSP 支持、内置浏览器（Agent 可直接访问本地网站）、插件市场（MCP、子 Agent）及重新设计的 Diff 视图。

**技术解读：**

Cursor 3 的架构转变意义深远：它不再是"你用 IDE，AI 来帮你"，而是"你调度 Agent 团队，IDE 是其中一个工具"。多仓库布局和本地云端无缝切换解决了大型项目中的 Agent 上下文边界问题。Composer 2 专为编码场景优化，强调速度与精准，避免了通用大模型在代码生成中的"过度解释"倾向。

**背景上下文：**

Cursor 自 2024 年以来持续引领 AI 编码工具浪潮，月活开发者突破百万。3.0 的发布是其与 GitHub Copilot Workspace、Replit Agent 等竞争对手在"Agent 编排"赛道上的正式宣战。

**开发者行动建议：**

- 升级 Cursor 后通过 `Cmd+Shift+P → Agents Window` 进入新界面
- 探索多仓库 Agent 并行场景，特别适合全栈项目（前后端分离仓库同步开发）
- 插件市场的 MCP 支持使 Cursor 可接入自定义工具链，值得优先评估
