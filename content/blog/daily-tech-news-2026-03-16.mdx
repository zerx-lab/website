### Mistral 发布 Leanstral：首个 Lean 4 形式化证明开源 Agent，成本是 Claude Sonnet 的 1/15 ⭐⭐⭐⭐⭐

**核心要点：**

- Mistral 发布 **Leanstral**（模型 ID：`labs-leanstral-2603`），这是首个专为 **Lean 4 形式化证明工程**设计的开源代码 Agent，权重采用 **Apache 2.0** 协议开放
- 在 pass@2 基准上，Leanstral 得分 **26.3**，超越 Claude Sonnet 2.6 分；在 pass@16 上达到 **31.9**，超越 Sonnet 8 分——而运行成本仅 36 美元，Sonnet 则需 549 美元（**约 1/15 的成本**）
- 采用**高度稀疏架构**，配合 Lean 作为完美验证器实现并行推理，兼顾性能与成本效率
- 同步发布评估套件 **FLTEval**，将形式化证明评估从竞赛数学扩展到更广泛的工程场景
- 已集成进 **Mistral Vibe** 零配置使用（通过 `/leanstall` 启动），并支持 lean-lsp-mcp 等任意 MCP 服务器；API 端点限时免费开放以收集反馈

**技术解读：**

形式化证明长期以来是 AI 辅助编程的"硬骨头"——Lean 4 代码需要类型系统完全正确才能通过编译器验证，不存在模糊的"看起来差不多对"。Leanstral 的突破在于将 Lean 编译器本身作为强化学习的即时验证信号，实现自我纠错的推理循环。对于构建证明库、验证算法正确性、或将数学定理形式化的开发者而言，这是目前最强的开源选项。

**背景上下文：**

这是 Mistral 继 Devstral 2 和 Mistral Vibe 2.0 之后，在编码 Agent 赛道持续加码的又一步棋，正面对标 Anthropic 的 Claude Code 生态。

**开发者行动建议：**

- 数学/证明方向：通过 Mistral Vibe 免费端点立即体验，限时无成本
- 在 CI 流程中引入 Lean 4 形式化验证的团队，可将 Leanstral API 集成为自动化证明生成步骤

### Bun v1.3.10：全新原生 REPL、浏览器编译目标、25x 更快的 structuredClone ⭐⭐⭐⭐

**核心要点：**

- Bun v1.3.10 修复 **155 个 issue**（累计 642 个 👍），核心亮点是用 Zig 完全重写的**原生 REPL**，支持语法高亮、top-level await、ESM import、Emacs 快捷键、Tab 补全与持久化历史记录
- 新增 `--compile --target=browser` 标志，可将 JS/CSS/Assets 打包为**单一自包含 HTML 文件**，无需构建服务器即可分发前端应用
- 新增 **Windows ARM64 原生支持**（Snapdragon 等），并可交叉编译针对 `bun-windows-arm64` 的独立可执行文件
- 性能提升显著：`structuredClone` 数组操作快 **25 倍**、`Buffer.slice()` 快 1.8 倍、`path.parse()` 快 7 倍、深层 Rope 字符串切片快 168 倍
- Barrel import 优化：自动检测纯重导出 index 文件（如 antd/index.js），只解析实际使用的子模块，大幅提升大型项目构建速度
- 修复 Python asyncio MCP 服务器与 `Bun.spawn()` 的兼容性问题

**技术解读：**

`--compile --target=browser` 是本次最具创新性的功能：将 Bun 的单文件可执行文件思路（原本用于服务端）延伸到浏览器端，一条命令输出可直接用浏览器打开的 HTML 包。对于构建内部工具、演示应用或离线 PWA 的团队，这消除了 CDN 和构建流水线的复杂性。

**开发者行动建议：**

- 运行 `bun upgrade` 立即获取 v1.3.10
- 尝试 `bun --compile --target=browser src/app.ts -o app.html` 体验单文件打包
